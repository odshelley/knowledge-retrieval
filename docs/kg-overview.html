1. What this is

kg lets you (and invited colleagues) ask research questions inside Claude Code and get answers grounded in the paper corpus — verbatim quotes, real citations, and an honest "the corpus doesn't cover this" when that's the truth. It consists of two pieces:

  • A hosted query server (this repo, server/): a stateless, read-only MCP server in front of the Neo4j Aura graph that the Dagster pipeline builds from raw PDFs. It exposes exactly 8 typed retrieval tools and holds all the secrets (Aura credentials, OpenAI key). Live at kg-graph.fly.dev.
  • A thin Claude Code plugin (~/Projects/kg, to be published as odshelley/kg): the ask skill (the intelligence — retrieval protocol + citation discipline), an .mcp.json pointing at the server, and a kg-setup token installer. Zero Python dependencies for installers.

The split is deliberate: the server is a dumb, safe query surface; the skill is where the reasoning lives. Colleagues hold only a personal bearer token — never database credentials.

The 8 tools

ToolWhat it does
search_chunksVector search over equation-aware paper chunks. expand="local" adds each hit's concepts/definitions/results/citations in one pass (GraphRAG local mode); expand="concepts" pivots retrieval to the concepts themselves (dual-level mode — catches same-idea-different-words).
search_papersFind papers by title or embedding relevance.
get_paperOne paper: metadata, authors, abstract/TLDR, structured Claude analysis.
get_conceptA concept with verbatim definitions (+ source papers) and related concepts.
get_resultsTheorems/lemmas/propositions/corollaries by concept or paper.
get_dependency_chainWalk Result DEPENDS_ON Result — theorem lineage across papers.
get_citationsCITES neighbourhood (who it cites / who cites it).
get_corpus_overviewCounts, top concepts, recent papers — the anti-hallucination scope check.

3. Design choices & why

DecisionChoiceRationale
Sharing modelHosted query serverKeys stay server-side; one shared corpus; colleagues need only a token. Rejected: shared DB creds (leaky), per-user graphs (heavy setup, no shared corpus).
ProtocolRemote MCP (streamable HTTP)Native Claude Code integration, cross-client standard, auth/distribution solved. Rejected: bespoke REST + client scripts (hand-rolled everything).
Query surfaceTyped tools onlyNo raw Cypher, no write tools, no "answer the question" tool. Predictable, safe, schema knowledge stays server-side; skills stay schema-ignorant.
HostingFly.io PaaS containerStateless service; always-on without your Mac; ~$2–3/mo; a Docker image you can lift anywhere later.
Repo layoutServer in knowledge-retrieval; plugin in its own public repoServer imports the pipeline's schema/embedding code directly (zero drift); pipeline repo stays private; plugin repo ships nothing sensitive.
AuthStatic per-user bearer tokens, salted-hashed server-sideRight-sized for a handful of trusted colleagues; constant-time verification; per-token rate limiting (60/min). OAuth is the upgrade path, not a v1 need.
Read-only, twiceDriver-level READ_ACCESS sessions + no write toolsEven a leaked token cannot modify the graph. A dedicated read-only DB user is the pending third layer.
Grounding rule“A claim without a retrieved chunk must be labelled interpretation”The single line that separates this from ordinary Claude. Enforced in the skill; verified by evals incl. an uncovered-topic honesty probe.

4. Gotchas we hit (so you don't)

  • FastMCP's hidden host filter. The MCP SDK enables localhost-only DNS-rebinding protection by default — every local test passed, and the first real deployment returned 421 Misdirected Request. Disabled via TransportSecuritySettings (bearer auth is the perimeter); a regression test now sends a deployed Host header.
  • The cloudflared tunnel owns port 8080. alethograph.com → localhost:8080, so a local test container on 8080 was briefly internet-reachable (auth held; scanners got 401s). Local testing now uses 127.0.0.1:8081. Don't park anything on 8080.
  • Skill grounding leaks. The ask skill once cited another installed skill's description as evidence of corpus coverage. Patched: coverage is judged only from kg tools, and the corpus overview (top concepts only) must be followed by a search probe before declaring a topic uncovered.
  • Sync I/O debt (accepted, ticketed). Tool calls block the event loop while querying Neo4j/OpenAI — fine at this scale, but the async offload + per-query timeouts + healthz cache should land before the user count grows.

6. Onboarding a colleague

A colleague needs exactly two things from you: a personal token, and the install instructions. They need Claude Code and nothing else — no Python packages, no database credentials, no OpenAI key.

Step 1 — you: mint a token & register it

cd ~/Projects/knowledge-retrieval          # (or the worktree)
uv run python scripts/issue_token.py ffion
# prints two lines:
#   token (give to colleague, do not store): kg_ffion_<32 hex>
#   KG_TOKENS entry (append server-side):    ffion:<salt>:<hash>

# append the entry to the server's secret (comma-separated with existing entries)
fly secrets set -a kg-graph KG_TOKENS="osian:...,ffion:<salt>:<hash>"

notefly secrets set replaces the whole value — always include the existing entries. Send the token (first line) over a private channel; you keep no copy (only the hash lives server-side). Lost token = mint a new one.

Step 2 — them: install

# in Claude Code
/plugin marketplace add odshelley/kg
/plugin install kg@kg
# restart Claude Code, then store the token:
! kg-setup kg_ffion_<their token>
# restart once more, then:
/kg:ask what does the corpus say about rectified flows?

pending This flow needs the plugin repo published first: gh repo create odshelley/kg --public --source ~/Projects/kg --push. Until then, a colleague can clone/receive the folder and use claude --plugin-dir path/to/kg.

Revoking access

fly secrets set -a kg-graph KG_TOKENS="<all entries except theirs>"

7. Daily use

  • With the plugin: /kg:ask <question> — the skill checks corpus scope, retrieves hybrid (vector + graph), and answers with [Author Year, chunk N] citations and a Sources list. Anything not backed by a retrieved chunk is labelled interpretation.
  • Tools-only in any session: claude mcp add --transport http --scope user kg https://kg-graph.fly.dev/v1/mcp --header "Authorization: Bearer <token>" — raw retrieval tools everywhere, without the skill's citation discipline. Remove this if you later install the plugin (duplicate server).
  • If it reports "server down": check /healthz first — it distinguishes a real outage from a client-side MCP drop (restart the session for the latter).

Roadmap (each: spec → independent review → plan → build; currently paused for v1 testing):

  • v2 — referee support: check a submitted paper against the corpus — prior art via CITES + concept overlap, novelty assessment, missing-citation list.
  • v2/v3 — paper-writing aid (high priority): strengthen arguments with citations, proof help via dependency chains, citation-correctness checking.
  • v3 — link discovery: cross-paper connections via shared concepts, dependency chains, and embedding proximity.
  • Enablers from the pipeline: topic inference (unlocks topic-subtree queries), book ingestion (book chunks appear in retrieval automatically).
  • Rename kg → alethograph once it replaces the old plugin.
